PAR Aircraft Synthesis Designer

Starts from the fuselage/core aircraft you have already built, using block-based dimensions and a required base fuselage mass plus a user-specified core CoG station, then synthesizes the wing, tail, fin, control surfaces, flap plan, mass growth, stability margin, trim state, and camber strategy needed to build a high-performing aircraft for the PAR aerodynamics mod in Space Engineers 1.

Inputs

Mass-Aware
Used for block size, structural block mass, rotor mass, hinge mass, and lost-block penalties. Changing grid converts the current block dimensions to preserve the same physical size.
Required. This is the mass of the aircraft before the designer adds wing, tail, fin, flaps, and control hardware.
Required. Enter the fuselage/core CoG station measured aft from the front edge or nose of the existing fuselage before wings and tails are added.
Measured from the fuselage bottom. The tail and elevator are assumed to stay at the fuselage top height; only the wing root height moves up or down.
Blank defaults to 200 m/s.
Advanced Assumptions
Default is 1.0 g, equivalent to Earthlike steady-flight sizing.
Approximate how streamlined the fuselage nose/leading body is. 0 = blunt brick, 100 = very clean slope-shaped body. Used only for fuselage form drag scaling.
Optional. Blank uses the aircraft-role default. The solver will try to improve wing AR, trim drag, and tail/control sizing toward this value before declaring the target unreachable.
Inputs do not recompute live while you edit them. The design updates only when you click Recompute Design.
